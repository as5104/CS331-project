Part B : 
Q1. Identify the key classes involved in your software. Name the classes
and identify each class's attributes and functionalities (methods).
Mention the visibility (public, private, protected) of the attributes
as well as the functions (methods) of each class.


1. User (Abstract Class)

Description:
The User class represents a generic system user. It is a base class
from which Student, Faculty, and Admin are derived.

Attributes:
+ userId : String
+ name : String
+ email : String
- password : String
+ role : String

Methods:
+ login() : boolean
+ logout() : void
+ viewDashboard() : void


2. Student (Inherits User)

Description:
The Student class represents a student who interacts with the system
to perform academic and administrative activities.

Attributes:
+ rollNo : String
+ department : String
+ cgpa : float

Methods:
+ submitAssignment() : void
+ applyForReevaluation() : void
+ submitLeaveRequest() : void
+ viewAttendance() : void
+ calculateCGPA() : float


3. Faculty (Inherits User)
