## Software Development Life Cycle (SDLC)

---

## 1. Requirement Analysis

The project begins with a thorough understanding of stakeholder needs.

**Key Stakeholders**
- Students  
- Faculty  
- Administrators  

**Identified Functional Requirements**
- Role-based authentication and login  
- Course registration and management  
- Attendance tracking  
- Grade and result management  
- Leave application and approval  
- University elections module  
- Notifications and announcements  

**Non-Functional Requirements**
- High security and data privacy  
- Scalability for large user bases  
- Real-time system access  
- Reliability and performance  

---

## 2. Feasibility Study

This phase evaluates the practicality of developing the system.

**Feasibility Aspects**
- Technical feasibility using web-based architecture  
- Use of open-source technologies  
- Cloud-based hosting support  
- Completion within the academic timeline  

The study confirms that the system is achievable with available resources and tools.

## 4. Development

The system is built according to the finalized design.

**Development Activities**
- Backend API development for academic and administrative operations  
- Frontend interface creation for different user roles  
- Integration of authentication and authorization  
- Use of version control systems for collaboration and tracking  

---

## 5. Testing

The system is tested to ensure reliability and correctness.

**Testing Focus Areas**
- Login and authentication  
- Course registration workflows  
- Attendance marking  
- Grade uploading and viewing  
- Role-based access validation  

**Testing Types**
- Unit testing  
- Integration testing  
- Bug fixing and refinement  

---

## 6. Deployment

After successful testing, the system is released for real-world use.

**Deployment Details**
- Frontend hosted on **Vercel**  
- Backend services deployed securely  
- Database configured with access controls  

The application is made accessible to students, faculty, and administrators.

---

## 7. Maintenance

Post-deployment support ensures long-term system effectiveness.

**Maintenance Activities**
- Bug fixes and issue resolution  
- Performance optimization  
- Academic data updates  
- Feature enhancements based on university needs
